# Simple Workflow

Use this skill when you need to make a small repository change and want to keep
the work organized. The workflow is intentionally plain so it can act as a
baseline fixture for compressors.

You MUST inspect the current files before editing. You SHOULD keep edits scoped
to the user request.

First, read the relevant files. Second, make the smallest useful change. Third,
run the relevant verification command. Fourth, summarize what changed.
